# Shift4 Payments — Competitive Map

## The Core Differentiation: Integrated vs. Pure Processing

Shift4's model: **software + hardware + payments in a single platform**. Revenue = toll on every transaction + software subscription. Switching costs = entire tech stack, not just a payment processor.

Pure processors (GPN, FIS, FISV): process transactions, no software. Easy to replace. This is why their growth is declining.

## FOUR vs. Legacy Processors (GPN, FIS, FISV)

| | Shift4 (FOUR) | Global Payments (GPN) | FIS | Fiserv (FISV) |
|--|--|--|--|--|
| Model | Software + Payments (integrated) | Pure processing + software bolt-ons | Pure processing + core banking | Pure processing + core banking |
| Revenue Growth | 29% GRLNF guided FY26 | Low-mid single digits | Low single digits | Low single digits |
| EBITDA Margin | ~46-51% | ~42% | ~34% | ~43% |
| EV/EBITDA | 6.0x | 9.3x | 12.4x | 7.1x |
| Organic Growth | Mid-teens | 5-6% | 3-4% | 4-5% |
| Switching Cost | Entire tech stack (POS, PMS, hardware) | Just payment rails | Just payment rails | Just payment rails |

**Why legacy processors are losing**: They bolted software onto payment processing. Shift4 built payments into software from day one. For hotels/stadiums/luxury retail, the software IS the business — swapping a POS means retraining staff, migrating data, and disrupting operations. Nobody does it.

## FOUR vs. Toast (TOST) — "Shift4 is the Anti-Toast"

| | Shift4 (FOUR) | Toast (TOST) |
|--|--|--|
| Verticals | Hotels (40% US), Stadiums (75%), Luxury retail, Restaurants, International | Restaurants ONLY |
| Market | Multi-vertical, enterprise + SMB | SMB restaurants, US-focused |
| Payments margin | ~62bps blended | ~61bps (similar) |
| EBITDA margin | ~46-51% adj. | ~5.8% |
| FCF generation | $500M+ | Minimal |
| International | 75+ countries via Global Blue | Negligible |
| M&A strategy | Diversification into adjacent verticals + geographies | None — organic only |
| Enterprise capability | Stadium complexes, resort networks, luxury retail chains | Not designed for it |

**Bottom line**: Toast wins restaurants. Shift4 wins everywhere else. These are NOT the same market. Toast is the gold standard for a 50-seat independent restaurant. Shift4 runs the Wynn casino resort. Different customers, different switching costs, different TAMs.

## FOUR vs. Modern Platforms (Adyen, Stripe, Block/SQ)

| | Shift4 (FOUR) | Adyen | Stripe | Block (SQ) |
|--|--|--|--|--|
| Focus | Venue/hospitality/luxury (physical) | Global enterprise e-commerce + unified | e-commerce + marketplaces | SMB physical retail |
| Physical POS strength | #1 in US hotels/stadiums | Strong in luxury retail (global chains) | Minimal POS | Strong SMB POS |
| DCC/TFS capability | Best-in-class via Global Blue | DCC offered | None | None |
| Integrated software | Deep (PMS, POS, ticketing, EMS) | Thin (payments-first) | Very thin (dev API-first) | SQ Dashboard only |
| Revenue model | Processing + software + DCC + TFS | Processing + platform fees | Processing + software | Processing + hardware |

**Key insight**: Adyen is the primary competitive threat in luxury retail enterprise — they've served global luxury chains for years. But Adyen doesn't do TFS/DCC at the depth Global Blue does, doesn't have stadium/hotel vertical software, and doesn't do SMB cross-sell. Shift4's all-in-one terminal (payments + DCC + TFS in one device) is genuinely differentiated.

## Shift4's Moat — 5 Layers

1. **Deep vertical software integration**: 28 years of PMS/POS integrations in hotels/stadiums. Not replicable quickly.
2. **Real switching costs**: Converting a 500-room hotel from Shift4 means replacing PMS, POS, hardware, all at once. Est. cost: $250K-$1M in transition + 6 months disruption.
3. **Blended spread durability**: 62bps held EVEN as Shift4 moved upmarket to larger clients. Pure processors compress; integrated platform providers don't.
4. **DCC + TFS stack**: Globally unique via Global Blue. No competitor offers payments + DCC + VAT refunds in a single handheld device across 75 countries.
5. **Network position in venue verticals**: ~75% of US stadiums means once a venue system is built around Shift4's VenueNext/SkyTab, it becomes the default. Stadiums don't switch often.
